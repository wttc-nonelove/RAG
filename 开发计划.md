|------|------|--------|------|

## 项目概况

本项目为课程设计项目，构建一套**基于检索增强生成（RAG）架构的私有知识库智能问答系统**。

| 维度 | 说明 |
|------|------|
| 技术栈 | Vue 3 + Element Plus（前端）、Python 3.11 + FastAPI（后端）、MySQL + ChromaDB + Neo4j + Redis（数据层） |
| LLM | DeepSeek API（默认），支持多供应商切换 |
| 功能规模 | 44 项功能需求，8 个模块（M1~M8） |
| 开发周期 | 6 个阶段 |

## Phase 1: 基础框架搭建

**目标**：搭建 Docker 开发环境、前后端脚手架、数据库建表、用户注册/登录/鉴权。

### 1.1 Docker 开发环境搭建

- 编写 `Docker-env/docker-compose.yml`，定义 6 个服务容器：

| 容器名 | 镜像 | 宿主机端口 | 容器端口 | 用途 |
|--------|------|-----------|----------|------|
| rag-mysql | mysql:8.0 | 3307 | 3306 | 关系数据库 |
| rag-redis | redis:7-alpine | 6379 | 6379 | 缓存 + Celery Broker |
| rag-neo4j | neo4j:4.4.16 | 7687, 7474 | 7687, 7474 | 图数据库 |
| rag-chromadb | chromadb/chroma:latest | 8001 | 8000 | 向量数据库 |
| rag-backend | (Dockerfile) | 8000 | 8000 | FastAPI 后端 |
| rag-frontend | (Dockerfile) | 5173 | 5173 | Vite 前端 |

- 编写 `Docker-env/mysql/init.sql`：完整 DDL（8 张表 + 索引）+ 预置数据 DML（admin 账号、系统配置项、DeepSeek 供应商/模型配置）
- 配置 `Docker-env/.env`：`DEEPSEEK_API_KEY`、`ENCRYPTION_KEY`、`JWT_SECRET_KEY` 等
- 验证：`docker compose up -d` → 所有容器 healthy → 服务间网络互通

### 1.2 后端脚手架

- 初始化 FastAPI 项目结构（按详细设计说明书目录结构）
- `app/main.py`：FastAPI 应用入口，注册 CORS 中间件，挂载路由
- `app/config.py`：全局配置（从环境变量读取数据库连接、API Key 等）
- `app/dependencies.py`：JWT 鉴权依赖注入（`get_current_user`，`require_admin`）
- 配置日志系统（`utils/logger.py`）
- 初始化：SQLAlchemy 异步引擎 + ChromaDB HttpClient + Neo4j driver + Redis client

### 1.3 前端脚手架

- `npm create vite@latest frontend -- --template vue` 初始化 Vue 3 项目
- 安装依赖：`element-plus`、`vue-router`、`pinia`、`axios`、`echarts`、`markdown-it`
- 配置 Vite 代理（`/api` → `http://backend:8000`）
- 搭建经典后台布局：`Sidebar.vue` + `Header.vue` + `<router-view>`
- 配置路由（10 条路由）和导航守卫（JWT 解析 + 角色校验）
- 封装 Axios 实例（`api/client.js`）：请求拦截器注入 Token，响应拦截器处理 401/5xx

### 1.4 数据库初始化

- MySQL：执行完整 DDL 创建 8 张表 + 索引 + 预置数据
- ChromaDB：验证 `kb_chunks` Collection 可正常创建
- Neo4j：验证 Entity 约束和索引创建成功
- Redis：验证连接和基础操作

### 1.5 用户认证模块（M5 核心）

后端文件：
- `models/user.py` — User ORM Model
- `schemas/auth.py` — 登录/注册/修改密码 Schema
- `dao/user_dao.py` — users 表 CRUD
- `services/user_service.py` — authenticate、register、create_user、reset_password、toggle_status、change_password
- `routers/auth.py` — POST login、POST register、GET me、PUT password
- `utils/encryption.py` — AES-256-CBC 加解密工具
- JWT 生成与验证（python-jose, HS256, 24h 过期）
- bcrypt 密码哈希（passlib, rounds=12）

前端文件：
- `views/Login.vue` — 登录表单 + 注册弹窗
- `stores/auth.js` — Pinia 认证状态
- 登录成功存储 Token → 按角色跳转（admin → /dashboard，user → /qa）

### 2.1 后端文档管理模块（M2）

- `dao/chroma_repo.py` — ChromaDB 操作封装（query、add_batch、delete_by_doc_id、count、health）
- `services/document_service.py` — upload_and_parse、parse_document（异步）、update_document（上传新版本，F-11）、get_preview（分页，F-12）、delete（级联清理 Vector + KG + 文件）

- `utils/embedding.py` — EmbeddingClient，调用当前默认 Embedding 模型
- 文档解析流水线：上传 → 文本提取（pdfplumber/python-docx/TXT） → 清洗 → 分块 → Embedding → 写入 ChromaDB

- `api/documents.js` — 7 个 API 封装函数
- 操作：上传、预览、上传新版本、删除（级联清理）、重新解析
- 标签管理：行内编辑标签（F-10）、标签筛选

## Phase 3: 智能问答核心

**目标**：RAG 检索链路、LLM 调用、多轮对话、答案溯源、用户模型选择、检索兜底。

### 3.1 后端 RAG 核心链路（M1）

- `models/conversation.py`、`models/message.py` — ORM Models
- `schemas/qa.py` — 问答请求/响应 Schema（含 fallback 响应格式）
- `dao/conversation_dao.py` — 会话 CRUD
- `dao/message_dao.py` — 消息 CRUD（含 get_recent_history）
- `dao/neo4j_repo.py` — Neo4j 操作封装（query_subgraph、get_overview、search 等）
- `utils/llm_client.py` — LLMClient 多供应商适配（OpenAI SDK 兼容模式），含重试逻辑
- `services/qa_service.py` — `rag_pipeline()`：
  1. 获取对话历史（最近 N 轮）
  2. 问题向量化（调用当前 Embedding 模型）
  3. ChromaDB 语义检索 Top-K
  4. 相似度判断：< threshold（默认 0.6）→ fallback，不调 LLM
  5. Neo4j 知识图谱补充（可选，受 kg_enabled 配置控制）
  6. 组装 Prompt（System Prompt 模板 + 上下文 + 对话历史）
  7. 调用对话模型生成回答
  8. 解析引用来源 → 保存问答记录至 MySQL
- `routers/qa.py` — 6 个接口（ask、conversations CRUD、messages、models）
- 检索兜底逻辑：`max_similarity < threshold` → 直接返回"知识库中暂无相关信息"，不调用 LLM

### 3.2 前端智能问答页面（M1）

- `views/user/QA.vue` — 问答主页面（左侧会话列表 + 右侧消息区 + 底部输入）
- `components/qa/ConvList.vue` — 会话列表（新建、切换、右键重命名/删除）
- `components/qa/ChatBubble.vue` — 聊天气泡（user/bot 样式区分，bot 消息含引用来源）
- `components/qa/SourceCard.vue` — 引用来源卡片（文档名 + 页码 + 原文摘要）
- `stores/qa.js` — Pinia 问答状态
- `api/qa.js` — 6 个 API 封装函数
- 模型选择下拉框（F-05）：从 API 获取可选模型，用户自行选择
- 多轮对话（F-02）：保持会话上下文，加载历史消息
- 答案溯源（F-03）：显示引用来源，点击查看原文
- 会话管理（F-04）：新建/切换/删除对话

## Phase 4: 知识图谱

### 4.1 后端 KG 抽取与管理（M3）

### 4.2 前端知识图谱页面（M3）

- `views/admin/Graph.vue` — 图谱管理主页面

  - 滚轮缩放（0.3x ~ 2.5x，以鼠标为中心）
  - 点击节点显示详情弹窗（类型、来源文档、描述）

## Phase 5: 管理功能完善

**目标**：工作台统计、系统配置、模型管理（多供应商/模型/Prompt 切换）、全局问答历史、用户管理。

### 5.1 用户管理（M5 剩余）

- 后端：`routers/users.py` — 5 个接口（列表、新增、编辑、重置密码、切换状态）
- 前端：`views/admin/Users.vue` — 用户列表（搜索、角色筛选、分页）、新增/编辑/禁用弹窗
- 前端：`views/user/Profile.vue` — 个人中心（修改密码）（F-28）

### 5.2 工作台（M6）

- 后端：`services/dashboard_service.py` — get_stats、get_trends、get_storage、get_system_status、get_doc_distribution
- 后端：`routers/dashboard.py` — 4 个接口
- 前端：`views/admin/Dashboard.vue` — 统计卡片 + ECharts 趋势折线图（7/30 天切换）+ 文档分布饼图 + 存储用量 + 系统状态表格

### 5.3 问答历史（M4）

- 后端：`services/history_service.py` — get_admin_history、get_my_history、get_stats
- 后端：`routers/history.py` — 4 个接口
- 前端：`views/admin/History.vue` — 全局问答历史（用户筛选、日期范围、关键词搜索、详情弹窗、统计卡片）
- 前端：`views/user/MyHistory.vue` — 个人问答历史

### 5.4 系统配置（M7）

- 后端：`services/config_service.py` — get_all、update_batch
- 后端：`routers/config.py` — 3 个接口
- 前端：`views/admin/Config.vue` — 滑块/开关表单（模型参数、检索参数、分块策略）+ 系统信息面板
- 配置项：temperature、top_p、max_tokens、top_k、similarity_threshold、chunk_size、chunk_overlap、kg_enabled、history_rounds

### 5.5 模型与 API 管理（M8）

- 后端：
  - `models/model_provider.py`、`models/model_config.py`、`models/model_preset.py` — ORM Models
  - `dao/model_dao.py` — providers/configs/presets CRUD
  - `services/model_service.py` — test_connectivity、set_default_model、save_prompt_template、create_preset/update_preset/delete_preset
  - `tasks/rebuild_tasks.py` — 全量向量重建异步任务
  - `routers/models.py` — 17 个接口（含预设管理）
- 前端：`views/admin/Model.vue` — 4 个 Tab：
  - **模型供应商**：添加/编辑/删除/连通性测试
  - **对话模型**：添加/编辑/设为默认/启用停用/Prompt 模板编辑
  - **Embedding 模型**：添加/编辑/设为默认（维度兼容检测 + 一键重建向量库入口）
  - **Prompt 模板**：System Prompt 编辑 + 上下文拼接格式 + 模板预设列表管理（F-41/F-43）
- Embedding 切换维度检测：维度相同 → 直接生效；维度不同 → 弹窗提示 → 一键重建向量库（F-44）
