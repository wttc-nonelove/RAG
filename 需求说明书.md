## 1. 项目背景与目标

面向企业用户，构建一套**基于检索增强生成（RAG）架构的私有知识库智能问答系统**。企业将内部文档导入知识库后，系统自动对文档进行解析、向量化、知识图谱构建，最终向终端用户提供基于私有知识的精准问答服务，降低信息检索成本，提升企业知识利用效率。

**核心目标**：
- 支持管理员便捷管理私有知识库（上传/解析/维护）
- 用户以自然语言提问，系统基于知识库内容返回精准、有出处的回答
- 知识图谱辅助推理，增强回答的关联性与完整性
- 支持灵活切换大模型及 Embedding 模型，适配不同 API 服务商
- 支持 Prompt 模板自定义，切换模型时自动适配最佳提问格式
- 用户可在问答页面自主选择模型，按需匹配不同问题场景

## 3. 业务流程

```
管理员
  │
  ├── 登录 → 上传知识库文档（PDF/Word/TXT）
  │           │
  │           ▼
  │         后台文档解析（pdfplumber / python-docx / TXT 原生读取）
  │           │
  │           ├──→ 文本分块 → Embedding（调用当前 Embedding 模型） → 存入 ChromaDB
  │           │
  │           └──→ LLM（调用当前对话模型）+spaCy 实体/关系抽取 → 存入 Neo4j 图数据库
  │
  ▼
普通用户
  │
  ├── 登录 → 输入自然语言问题
  │           │
  │           ▼
  │         RAG 流程：
  │           1. 问题向量化 → ChromaDB 相似度检索 Top-K 相关文本块
  │           2. Neo4j 补充关联知识（可选）
  │           3. 拼接 Prompt → 调用 DeepSeek-V4 生成回答
  │           4. 返回答案 + 引用出处
  │           │
  │           ▼
  │         用户查看回答（可继续追问 / 查看历史）
  │
  └── 问答记录存入 MySQL（供历史查看与统计分析）
```

## 4. 功能需求

### 4.1 智能问答模块（核心）

| 编号 | 功能 | 描述 | 用户角色 |
|------|------|------|----------|
| F-01 | 自然语言提问 | 用户输入问题，系统基于知识库生成精准回答 | 普通用户 |
| F-02 | 多轮对话 | 支持上下文关联的连续追问，保持会话状态 | 普通用户 |
| F-03 | 答案溯源 | 回答附引用来源（原文片段 / 文档名称 / 页码），支持点击查看原文 | 普通用户 |
| F-04 | 会话管理 | 用户可新建对话、切换历史对话、删除对话 | 普通用户 |
| F-05 | 用户模型选择 | 问答页面提供模型切换下拉框，用户可在管理员启用的模型范围内自行选择当次使用的对话模型 | 普通用户 |

**问答流程**：
```
用户输入问题
    → 问题向量化（调用当前选中的 Embedding 模型）
    → ChromaDB 检索 Top-K 相关片段
    → 相似度判断：
        ├── 最高相似度 ≥ 阈值（默认 0.6）→ 继续后续流程
        └── 最高相似度 < 阈值 → 直接返回提示"知识库中暂无相关信息，请尝试更换问题或补充相关文档"，不调用 LLM，流程结束
    → Neo4j 查询关联知识节点（可选）
    → 组装 Prompt（System Prompt 模板 + 上下文片段 + 对话历史 + 用户问题）
    → 调用当前选中的对话模型生成回答
    → 返回回答 + 引用来源
    → 记录问答日志至 MySQL
```

### 4.2 知识库管理模块

| 编号 | 功能 | 描述 | 用户角色 |
|------|------|------|----------|
| F-06 | 文档上传 | 支持上传 PDF、DOCX、TXT 文件，支持拖拽上传与批量上传 | 管理员 |
| F-07 | 文档列表 | 展示已上传文档，含文件名、类型、大小、解析状态、上传时间，支持搜索/筛选 | 管理员 |
| F-08 | 文档删除 | 管理员可删除文档，同步清理 ChromaDB 向量与 Neo4j 图数据 | 管理员 |
| F-09 | 解析状态追踪 | 实时展示文档解析进度（待解析 / 解析中 / 已完成 / 解析失败），支持手动重新解析失败文档 | 管理员 |
| F-10 | 知识库分类 | 支持按标签/分类管理文档（如：制度文件、技术文档、培训资料等） | 管理员 |
| F-11 | 文档更新 | 管理员在文档列表中选择目标文档，点击"上传新版本"，选择新文件后系统以新文件替换旧文档；覆盖时自动清理旧文档在 ChromaDB 的向量和 Neo4j 的图谱数据，重新解析新文件入库，保留问答历史中的旧引用链接（标记为"已更新版本"），documents 表 version 字段自增 | 管理员 |
| F-12 | 文档预览 | 管理员可在线预览已上传文档内容（PDF/DOCX/TXT），无需下载到本地，支持分页浏览与文本搜索 | 管理员 |

**文档解析流水线**：
```
上传文档（PDF/DOCX/TXT）
    → pdfplumber / python-docx / TXT 原生读取 提取文本
    → 文本清洗（去噪、分段、去重）
    → 文本分块（Chunk）策略：固定长度 + 重叠滑动窗口
    → Embedding 向量化（调用当前选中的 Embedding 模型）
    → 存入 ChromaDB（含元数据：文档 ID、文件名、页码等）
    → LLM（调用当前选中的对话模型）+ spaCy 实体/关系抽取
    → 存入 Neo4j 图数据库
    → 更新解析状态为"已完成"
```

**知识图谱抽取策略**：
- 文档上传后全自动抽取，无需管理员审核，直接入库 Neo4j

- 抽取类型：实体（术语/概念/人名/地名/日期/组织/产品名等）、关系（属于/包含/依赖/等同于/前置于/关联等）

| F-38 | 对话模型切换 | 从已配置的供应商中选择当前使用的对话模型（如 deepseek-chat、gpt-4o、qwen-max 等），支持动态切换，即时生效 | 管理员 |

| F-41 | 模型预设管理 | 系统内置常见模型预设（DeepSeek 系列、OpenAI 系列等），管理员可一键选用或基于预设修改 | 管理员 |
| F-42 | 默认模型设置 | 指定系统默认的对话模型和 Embedding 模型，新会话自动使用默认模型 | 管理员 |

| F-44 | Embedding 向量重建 | 切换 Embedding 模型后，系统检测向量维度不兼容并提示"已有向量与新模型不兼容，需全量重建向量库"，提供一键重建入口；管理员可手动设定目标向量维度 | 管理员 |

**模型切换配置结构**：
```
模型供应商 (Provider)
  ├── 供应商名称（如 DeepSeek）
  ├── API Base URL（如 https://api.deepseek.com/v1）
  ├── API Key
  └── 模型列表
        ├── 对话模型：deepseek-chat, deepseek-reasoner 等
        │     └── 专属 Prompt 模板：System Prompt + 上下文拼接格式
        └── Embedding 模型：deepseek-embedding 等
              └── 向量维度：由管理员设定或模型自动返回
```

**模型切换流程**：
```
管理员在"模型管理"页面
    → 选择目标模型供应商
    → 填写 API Base URL + API Key
    → 点击"连通性测试"验证可用性
    → 勾选需启用的对话模型 / Embedding 模型
    → 为每个对话模型配置 System Prompt 模板
    → 设为默认模型 → 保存
    → 若切换了 Embedding 模型，系统检测维度兼容性：
        ├── 维度相同 → 直接生效
        └── 维度不同 → 弹窗提示"已有向量与新模型不兼容，需全量重建向量库"
              → 管理员点击"一键重建" → 后台异步重建所有文档向量
    → 后续问答 / 文档解析自动使用新模型
```

## 7. 数据库设计概要

### 7.1 MySQL（关系型）

| model_configs | id, provider_id, model_name, model_type, system_prompt, embedding_dimension, is_default, is_active, created_at | 模型配置表（chat/embedding），含 Prompt 模板和向量维度 |

**model_providers 表详细字段**：

| 字段 | 类型 | 说明 |

| api_base_url | VARCHAR(255) | API 基础地址 |
| api_key_encrypted | TEXT | AES 加密存储的 API Key |
| is_active | BOOLEAN | 是否启用，默认 TRUE |

**model_configs 表详细字段**：

| 字段 | 类型 | 说明 |
|------|------|------|

| 14 | 用户模型选择 | 问答页面提供模型下拉框，用户可在管理员启用的模型范围内自行选择 |
| 15 | 文档更新 | 支持上传新版本覆盖旧文档，自动清理旧向量和 KG 数据，保留历史引用 |
| 16 | 文档预览 | 管理员可在线预览已上传文档内容（PDF/DOCX/TXT），支持分页浏览与文本搜索 |
| 17 | 检索兜底 | 相似度低于阈值时不调用 LLM，直接提示"知识库中暂无相关信息" |
| 18 | 文档更新方式 | 管理员在列表中选择目标文档 → 点击"上传新版本" → 选择新文件替换 |
