## 1. 系统概述

### 1.1 系统简介

本系统是一个基于 RAG（Retrieval-Augmented Generation，检索增强生成）架构的私有知识库智能问答系统。用户上传文档后，系统自动解析、向量化并建立知识图谱，通过 AI 大模型基于知识库内容回答用户问题。

### 1.2 技术架构

```
┌─────────────┐     ┌──────────────┐     ┌───────────┐
│  Vue 3 前端  │────→│ FastAPI 后端  │────→│ DeepSeek  │
│  :5173      │     │  :8000       │     │  API      │
└─────────────┘     └──────┬───────┘     └───────────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
        ┌─────┴─────┐ ┌───┴───┐ ┌─────┴─────┐
        │  MySQL    │ │ChromaDB│ │  Neo4j    │
        │  :3307    │ │ :8001  │ │  :7474    │
        └───────────┘ └───────┘ └───────────┘
```

| 组件 | 作用 | 端口 |
|------|------|------|
| rag-frontend | 用户界面 | 5173 |
| rag-backend | API 服务 | 8000 |
| rag-mysql | 用户/文档/配置存储 | 3307 |
| rag-chromadb | 文档向量存储 | 8001 |
| rag-neo4j | 知识图谱存储 | 7474 |
| rag-redis | 缓存 | 6379 |

### 1.3 默认账户

| 角色 | 用户名 | 密码 |
|------|--------|------|
| 管理员 | admin | admin123 |

## 2. 环境部署

### 2.1 前置要求

- Docker Desktop 已安装并运行
- 至少 4GB 可用内存
- 端口 5173、8000、3307、6379、7474、7687、8001 未被占用

### 2.2 启动系统

```bash
# 进入项目目录
cd F:\课设\资料\RAG2.0\docker

# 启动所有服务
docker compose up -d --build

# 查看运行状态
docker compose ps
```

所有 6 个容器状态应为 `Up` 或 `healthy`。

### 2.3 停止系统

```bash
docker compose down
```

### 2.4 查看日志

```bash
# 查看后端日志
docker compose logs rag-backend -f

# 查看前端日志
docker compose logs rag-frontend -f
```

## 3. 系统登录

### 3.1 访问地址

- 智能问答 - AI 对话

## 4. 智能问答（用户端）

### 4.1 新建对话

1. 点击左侧「新建对话」按钮
2. 系统自动创建新会话并切换到对话界面
3. 无需二次点击

### 4.2 发送问题

1. 在底部输入框输入问题
2. （可选）从下拉框选择对话模型，默认使用系统配置的默认模型
3. 点击「发送」或按 Enter 键
4. 等待 AI 回答（通常 2-5 秒）

### 4.3 对话交互

- AI 回答支持 Markdown 格式（标题、列表、加粗、代码块等）
- 回答底部显示引用来源标签（来自哪个文档、相似度分数）
- 支持多轮对话，系统自动携带历史上下文

### 4.4 会话管理

- 左侧列表显示所有历史会话
- 点击会话标题切换对话
- 点击会话右侧红色删除图标删除会话

## 5. 知识库管理

### 5.1 上传文档

1. 点击右上角「上传文档」按钮
2. 弹出上传对话框，支持以下方式选择文件：
   - 拖拽文件到上传区域
   - 点击上传区域浏览选择
3. 支持同时选择多个文件
4. （可选）填写标签用于分类
5. 点击「开始上传」
6. 上传队列中每个文件显示独立状态：
   - 待解析 - 文件已选择，等待上传
   - 上传中 - 正在上传到服务器
   - 已完成 - 上传并解析成功
   - 失败 - 上传或解析失败

**支持格式：** PDF、DOCX、TXT

### 5.2 文档列表

- 默认显示全部文档，按上传时间倒序
- 支持按关键词搜索文件名
- 支持按文件类型筛选（PDF / DOCX / TXT）
- 支持按状态筛选（pending / parsing / completed / failed）
- 每页显示 20 条，底部翻页

### 5.3 文档操作

| 操作 | 说明 |
|------|------|
| 预览 | 分页查看文档文本内容 |
| 更新 | 上传新版本文件替换原文档 |
| 重新解析 | 重新执行文档解析、向量化、知识图谱提取 |
| 删除 | 删除文档及其向量数据和知识图谱关联 |

### 5.4 文档解析流程

```
上传文件 → 保存到服务器 → 提取文本 → 文本清洗 → 分块 → 向量化 → 存入 ChromaDB
                                                         ↓
                                                    实体/关系提取 → 存入 Neo4j
```

- 解析状态 `parsing` 表示正在处理
- 解析状态 `completed` 表示可被问答检索
- 解析状态 `failed` 可点击「重新解析」重试

## 6. 知识图谱

### 6.1 图谱浏览

- 进入页面自动加载全量图谱
- 鼠标滚轮缩放，拖拽平移画布
- 拖拽节点调整位置
- 节点按实体类型着色（参见顶部图例）
- 悬停节点/边显示详情

### 6.2 搜索

1. 在搜索框输入关键词
2. 点击「搜索」或按 Enter
3. 显示包含关键词的实体及其关联关系
4. 点击「全部」恢复全量视图

### 6.3 添加实体

1. 点击「添加实体」按钮
2. 填写表单：
   - 名称：实体名称（必填）
   - 类型：从下拉框选择（人物/组织/地点/概念/技术/产品/事件）
   - 描述：实体描述（可选）
3. 点击「确定」
4. 图谱自动刷新

### 6.4 添加关系

1. 点击「添加关系」按钮
2. 填写表单：
   - 源实体：从已有节点下拉选择
   - 关系：关系描述（如「属于」「包含」「依赖」）
   - 目标实体：从已有节点下拉选择
3. 源实体和目标实体不能相同
4. 点击「确定」

### 6.5 删除实体

1. 点击要删除的节点
2. 弹出操作面板，显示实体名称
3. 点击「删除实体」按钮
4. 二次确认后删除该实体及其所有关联关系

## 7. 模型管理

模型管理页面包含 4 个标签页：模型供应商、对话模型、Embedding 模型、Prompt 模板。

### 7.1 供应商管理

#### 查看供应商
- 进入页面默认显示供应商列表
- 显示供应商名称、API 地址、状态

#### 添加供应商
1. 点击「添加供应商」
2. 填写：
   - 供应商名称（如 DeepSeek、通义千问）
   - API 地址（如 `https://api.deepseek.com/v1`）
   - API Key
3. 点击「确定」

#### 编辑供应商
1. 点击供应商行的「编辑」按钮
2. 修改名称、API 地址或 API Key（Key 留空则不更新）
3. 点击「确定」

#### 测试连通性
- 点击「测试」按钮
- 系统向供应商 API 发送请求
- 成功显示延迟毫秒数，失败显示错误信息

#### 删除供应商
- 点击「删除」，二次确认后删除
- 注意：删除供应商会级联删除其下的所有模型配置

### 7.2 对话模型

#### 添加模型配置
1. 切换到「对话模型」标签
2. 点击「添加模型」
3. 填写：
   - 供应商：从下拉框选择已添加的供应商
   - 模型名称：如 `deepseek-chat`、`deepseek-reasoner`
   - 系统提示词：可选，自定义模型的角色设定
   - 设为默认：开启后该模型为 QA 默认使用模型
4. 点击「确定」

#### 设为默认
- 点击模型行的「设为默认」按钮
- 同类型中只有一个模型可设为默认
- QA 页面未选择模型时自动使用默认模型

#### 编辑模型
- 点击「编辑」修改供应商、名称、提示词等

### 7.3 Embedding 模型

操作方式与对话模型类似，额外包含：
- **Embedding 维度**：向量维度参数（如 1536）
- 添加远程 Embedding 模型后，文档解析和问答检索将使用该模型生成向量

### 7.4 Prompt 模板

- 显示所有对话模型的系统提示词
- 每个模型独立的文本框，可直接编辑
- 点击「保存」更新
- 该提示词会作为 RAG 问答时的系统指令前缀

## 8. 问答历史

### 8.1 历史列表

- 显示所有用户的问答记录
- 包含：ID、问题摘要、用户、时间

## 10. 系统配置

### 10.1 检索参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| top_k | 检索返回的最大文档片段数 | 5 |
| similarity_threshold | 相似度阈值，低于此值的结果不纳入上下文 | 0.6 |
| history_rounds | 携带的历史对话轮数 | 5 |

### 10.2 生成参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| temperature | 生成温度，越高越随机 | 0.7 |
| top_p | 核采样参数 | 0.9 |
| max_tokens | 最大生成 token 数 | 2048 |

### 10.3 功能开关

| 参数 | 说明 | 默认值 |
|------|------|--------|
| kg_enabled | 是否启用知识图谱增强 | true |

## 11. 常见问题

### Q: 文档上传后状态显示 failed

**A:** 点击「重新解析」重试。如果持续失败，检查后端日志：
```bash
docker compose logs rag-backend --tail 50
```

### Q: 问答回答「知识库中暂无相关信息」

**A:** 可能原因：
1. 文档未完成解析（状态不是 completed）
2. 相似度阈值设置过高，尝试降低 `similarity_threshold`
3. 问题与文档内容关联度低，尝试换一种问法

### Q: 知识图谱页面为空

**A:** 确认：
1. 已上传文档且状态为 completed
2. 文档内容中包含可提取的实体
3. Neo4j 服务正常运行（`docker compose ps` 查看）

### Q: 模型测试显示失败

**A:** 检查：
1. API Key 是否正确（供应商列表 → 编辑 → 更新 Key）
2. API 地址是否可访问
3. 账户余额是否充足

### Q: 切换 Embedding 模型后检索不准

**A:** 切换 Embedding 模型后，已有的向量数据使用旧模型生成，需要重新解析所有文档：
1. 进入知识库管理
2. 对每个文档点击「重新解析」
3. 等待状态变为 completed

### Q: Docker 容器启动失败

**A:**
```bash
# 查看失败原因
docker compose ps
docker compose logs <服务名>

# 常见原因：端口占用
netstat -ano | findstr ":5173"

# 重建容器
docker compose down
docker compose up -d --build
```

### Q: 前端页面无法访问

**A:**
1. 确认 rag-frontend 容器运行中：`docker compose ps`
2. 确认访问地址：`http://localhost:5173`（不是 https）
3. 清除浏览器缓存后重试
